<p>Sometimes, instead of listing a set of values for compliance with a condition, we are offered a range of values. In this case, it is not possible to use the operator <code>IN</code>, since the list of possible values could become infinite in the case, for example a range of real numbers.</p><p>On those occasions we will use the range operator <code>BETWEEN</code>.</p><h3>Example:</h3><p>The following consultation shows the data of employees who have a wage between 9000 and 10000, both included:</p><blockquote><p><code>SELECT FIRST_NAME, LAST_NAME, SALARY<br/>FROM employees<br/>WHERE SALARY BETWEEN 9000 AND 10000<br/>ORDER BY SALARY DESC, LAST_NAME;</code></p></blockquote><table align="center" border="1" summary="Results from query 23"><tbody><tr><th style="background-color:#9e9e9e; text-align:center">FIRST_NAME</th><th style="background-color:#9e9e9e; text-align:center">LAST_NAME</th><th style="background-color:#9e9e9e; text-align:center">SALARY</th></tr><tr><td>Herman</td><td>Baer</td><td>10000</td></tr><tr><td>Harrison</td><td>Bloom</td><td>10000</td></tr><tr><td>Janette</td><td>King</td><td>10000</td></tr><tr><td>Peter</td><td>Tucker.</td><td>10000</td></tr><tr><td>Tayler</td><td>Fox</td><td>9600</td></tr><tr><td>David</td><td>Bernstein</td><td>9500</td></tr><tr><td>Danielle</td><td>Greene</td><td>9500</td></tr><tr><td>Patrick.</td><td>Sully</td><td>9500</td></tr><tr><td>Daniel</td><td>Faviet</td><td>9000</td></tr><tr><td>Peter</td><td>Hall</td><td>9000</td></tr><tr><td>Alexander</td><td>Hunold</td><td>9000</td></tr><tr><td>Allan</td><td>McEwen</td><td>9000</td></tr></tbody></table><h3>Exercise:</h3><p>It shows the name, surnames and date of recruitment (<code>HIRE_DATE</code>) of employees who were recruited between 1 January and 31 December 2000.</p><p>For this exercise you must handle dates, which is a special type of data that we have not yet used, but whose use is very common.</p><p>Depending on the server settings, the date format can change.
