<p>Nós dissemos que o resultado de uma composição interna nós podemos lidar com isso como outra tabela. Especificamente, o resultado de uma composição interna pode fazer parte de uma composição interna maior com outra tabela e assim por diante.</p><p>Podíamos ter, na cláusula. <code>FROM</code>uma composição como a seguinte:</p><p><code>FROM tabla1 JOIN tabla2 ON (tabla1.atributo1 = tabla2.atributo2) JOIN tabla3 ON (tabla2.atributo2 = tabla3.atributo3)</code></p><h3>Exemplo:</h3><p>A seguinte frase mostra os departamentos que estão no país cujo nome é 'Reino Unido'.</p><blockquote><p><code>SELECT DEPARTMENT_NAME, COUNTRY_NAME<br/>FROM departments JOIN locations USING (LOCATION_ID) JOIN countries USING (COUNTRY_ID)<br/>WHERE COUNTRY_NAME = 'United Kingdom';</code></p></blockquote><p>Devemos notar que, neste caso, temos usado <code>USING</code> porque a relação que estamos fazendo entre dois atributos que têm o mesmo nome, embora possamos também ter usado <code>ON</code>:</p><blockquote><p><code>ON (departments.LOCATION_ID = locations.LOCATION_ID)</code></p></blockquote><p></p><table align="center" border="1" summary="Results from query 18"><tbody><tr><th style="text-align: center; background-color: rgb(158, 158, 158);">DEPARTAMENTO_NAME</th><th style="text-align: center; background-color: rgb(158, 158, 158);">PAÍSES</th></tr><tr><td>Recursos humanos</td><td>Reino Unido</td></tr><tr><td>Vendas</td><td>Reino Unido</td></tr></tbody></table><p>Embora no <code>SELECT</code> Usamos apenas atributos de tabelas <code>departments</code> e <code>countries</code>No <code>FROM</code> Precisamos também incluir a tabela <code>locations</code> porque não é possível alcançar <code>countries</code> a partir de <code>departments</code>Sem passar <code>locations</code>.
