<p>Once the Cartesian product or the internal combination has been obtained, we can add restrictions to the result as if it were another table. I mean, we can:</p><ul><li>restrict the number of attributes we want to show in the result.</li><li>add restrictions to the clause <code>WHERE</code>.</li></ul><p><strong>OK!</strong> A very common error is to use, in a multi-fold query, the name of an attribute that is repeated in more than one table. The result is the one shown:</p><blockquote><p><code>SELECT <strong>REGION_ID</strong>, COUNTRY_NAME<br/>FROM regions, countries<br/>WHERE<br/>    regions.REGION_ID = countries.REGION_ID<br/>    AND<br/>    REGION_NAME = 'Europe';</code></p></blockquote><p><code><strong>ERROR:</strong>  column reference "region_id" is ambiguous</code></p><p>When that happens, we must point out which of the boards we want to obtain that value, by applying it to the name of the attribute and by separating it by a <code>.</code>:</p><blockquote><p><code>SELECT <strong>regions.REGION_ID</strong>, COUNTRY_NAME<br/>FROM regions, countries<br/>WHERE<br/>    regions.REGION_ID = countries.REGION_ID<br/>    AND<br/>    REGION_NAME = 'Europe';</code></p></blockquote><table align="center" border="1" summary="Results from query"><tbody><tr><th style="background-color:#9e9e9e;
