<p>Quando nos referimos a atributos de uma tabela, deve ser feito com o nome exato com o qual aparece na definição dessa tabela.</p><p>Anteriormente, temos relacionados os dados que queremos obter, em uma consulta sobre os funcionários da tabela, com os nomes dos atributos a que temos de referir:</p><ul><li>nome = 2005 <code>FIRST_NAME</code></li><li>Sobrenome = <code>LAST_NAME</code></li><li>trabalho = educação <code>JOB_ID</code></li></ul><p>Então, usámos a sentença.</p><blockquote><p><code>SELECT FIRST_NAME, LAST_NAME, JOB_ID<br/>FROM employees;</code></p></blockquote><p>e obtivemos, como resultado:</p><table align="center" border="1" summary="Results from query 3"><tbody><tr><th>FIRST_NAME</th><th>LAST_NAME</th><th>JOB_ID</th></tr><tr><td>William.</td><td>Gietz</td><td>AC_ACCOUNDO</td></tr><tr><td>Shelley.</td><td>Higgins</td><td>AC_MGR</td></tr><tr><td>Jennifer.</td><td>Baleias</td><td>AD_ASST</td></tr><tr><td>Steven.</td><td>Rei.</td><td>AD_PRES</td></tr><tr><td>Querida.</td><td>Kochhar</td><td>AD_VP</td></tr></tbody></table><p>No entanto, o que podemos fazer é facilitar a leitura dos dados para o usuário, mostrando como cabeçalhos de coluna da tabela o texto que queremos.</p><p>É para isso que os aliases da coluna são usados.</p><h3>Exemplo:</h3><p>A sentença</p><blockquote><p><code>SELECT FIRST_NAME AS "Name", LAST_NAME AS "Surname", JOB_ID AS "Job"<br/>FROM employees;</code></p></blockquote><p>Ele mostra os mesmos dados no conteúdo da tabela, mas os cabeçalhos dessas tabelas foram especificados no julgamento, em vez de usar os nomes dos atributos:</p><table align="center" border="1" summary="Results from
