<p>When we refer to attributes of a table, it must be done with the exact name with which it appears in the definition of that table.</p>
<p>Previously, we have related the data we want to obtain, in a consultation on the table employees, with the names of the attributes to which we have to refer:</p>
<ul><li>name = 2005 <code>FIRST_NAME</code></li><li>last name = <code>LAST_NAME</code></li><li>work = education <code>JOB_ID</code></li></ul>
<p>So, we used the sentence.</p><blockquote><p><code>SELECT FIRST_NAME, LAST_NAME, JOB_ID<br/>FROM employees;</code></p></blockquote>
<p>and we obtained, as a result:</p>

<p>However, what we can do is facilitate the reading of the data to the user, showing as column headers of the table the text we want.</p>
<p>That's what column aliases are used for.</p><h3>Example:</h3><p>The sentence</p>
<blockquote><p><code>SELECT FIRST_NAME AS "Name", LAST_NAME AS "Surname", JOB_ID AS "Job"<br/>FROM employees;</code></p></blockquote>
<p>It shows the same data in the contents of the table, but the headings of these tables have been specified in the judgment, instead of using the names of the attributes:</p>
